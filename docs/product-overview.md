# MyTrack Home 产品文档（品牌定位版）

## 1. 产品定位

MyTrack Home 是 **个人品牌官网**，不是某个单一子产品官网。

- 主体: MyTrack（品牌层）
- 子产品: iTrackDaily（当前已上线）
- 目标: 对外表达品牌方法论，并承接未来产品矩阵

## 2. 核心定义

- 品牌角色: 后续产品开发的统一理念与质量标准
- 子产品角色: 品牌方法论在具体场景中的实现
- 当前品牌主张: 隐私优先、清晰反馈、低负担日常价值

## 3. 信息架构

- `/` 首页（品牌定位 + 子产品入口）
- `/products` 产品矩阵（当前 + 未来方向）
- `/philosophy` 产品理念与开发约束
- `/about` 品牌背景
- `/contact` 联系

## 4. 设计策略

- 视觉方向: 蓝白清爽、专业克制
- 语气: 极简专业 + 温暖亲和
- 交互策略: 首屏快速理解品牌，再引导到子产品体验

## 5. 关键业务逻辑

- `daily.mytrack.club` 仅作为子产品外链入口
- 官网承担品牌解释与产品矩阵承接，不承担子产品全部说明
- 产品理念页用于沉淀未来产品开发原则

## 6. 工程现状

- React + TypeScript + Vite
- 双语 `en/zh`
- 路由由 `src/App.tsx` 统一管理
- 文案由 `src/content/siteContent.ts` 统一管理

## 7. 后续建议

1. 保持品牌叙事稳定，子产品信息只做“入口级”展示。
2. 新子产品上线时，先在 `/products` 扩展，再做独立落地页。
3. 把“理念 -> 产品 -> 用户价值”做成固定叙事模板复用。
